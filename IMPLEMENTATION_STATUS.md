# PUNI Life Implementation Status

## Complete

- Project folder renamed to `punilife`.
- Legacy prototype files and references removed.
- Unity 6.4 project opened and compiled once.
- Product name, bundle ID, Android orientation, and version settings added.
- Runtime bootstrap creates Camera, UI, `GameManager`, and Mock `AdManager`.
- Core data model: status, hidden growth stats, save data, stage, evolution type, dex entry.
- JSON save/load with backup and temp-file write path.
- Offline progress based on `lastSavedAt`.
- Care actions: Feed, Play, Clean, Sleep, Study, Train.
- Growth stage flow: Egg, Baby, Young, Evolved.
- Evolution routes: Sunny, Scholar, Brave, Forest, Shadow.
- Dex/Garden MVP panel.
- Snack Tap minigame with timer, score, combo, result panel.
- Mock rewarded ad placements.
- Debug panel for fast local validation.

## Partial

- UI is runtime-generated MVP UI, not final art/prefab UI.
- PUNI character is procedural placeholder art.
- Garden restoration is text/status based, not visual background progression.
- Shop item data exists, but no shop UI or item purchase flow yet.
- Ads are Mock only, not AdMob SDK.
- Scenes are documented, but dedicated scene assets are not yet authored.

## Not Started

- Real PUNI artwork and animation.
- Audio and vibration.
- Real AdMob integration.
- Android device build/test.
- iOS build support and Xcode export.
- Splash screen and store metadata.
- Supabase, ranking, cloud save, events.

## Known Technical Notes

- Empty scenes are supported through `PuniLifeBootstrap`.
- `Library/` and `Logs/` are generated by Unity and ignored by git.
- `Assembly-CSharp.dll` was generated by Unity 6.4, indicating C# compilation succeeded at least once.
- Reopen Unity after Codex edits to regenerate `.meta` files for new scripts and recompile.
